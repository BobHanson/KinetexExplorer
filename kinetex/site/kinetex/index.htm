Kinetics Explorer Concept Outline

to concept page

 

What is the “Kinetics Explorer” concept? The basic idea of this project is to provide an on-line resource to accompany discussions relating to chemical kinetics. The Kinetics Explorer is designed to be used either as an auxiliary to a book chapter on kinetics or as a stand-alone “electronic book.”

 

Why on-line? The rationale for choosing the web as a medium for delivery has to do with the nature of kinetics itself. Kinetics has to do with the dynamic time-dependent nature of chemical reactions. The Kinetics Explorer uses JavaScript-based dynamic HTML to simulate all of the important aspects of chemical kinetics in real time. “Reactions” change color in accord with the absorption of light of their component reactants and products. Data are “collected” and displayed in real time as tables and graphs of concentration vs. time. Multiple data sets can be displayed together or side-by-side to make direct comparisons. Exploration of the data begins with clicking on a graphed data point or selecting a set of data points to display a tangent line and automatically determine the slope at that point or over that range.

 

How is this “discovery learning”? The Kinetics Explorer provides a context for hands-on discovery learning. The logical sequence is from empirical data to theoretical understanding. Each of the “explorations” is introduced with a short discussion of the key concepts that will be important. Each then concludes with a few questions that allow the student to self-assess their level of mastery over the material.

 

Why JavaScript? From a development point of view, Kinetics Explorer was designed to be very flexible. Six component parts compose the system:

 

Model              the underlying theoretical model and variable for the reaction

            Reaction           an optional visual display of a reaction “flask”

            Experiment       the underlying experimental run

            Data                 collected data relating to concentration and time

            Graph               a real-time graph of the data using Cascading Style Sheet technology

            Analysis            a set of tools for displaying slopes and determining rates

 

Each of these component parts is optionally displayed or hidden. This allows the Kinetics Explorer package to be seen in a variety of detail or be completely behind the scenes. While the underlying JavaScript is complex, the implementation on a web page is almost trivial: Define the model using a few simple JavaScript function calls, select what is to be displayed from a limited set of options, arrange the elements on a web page using HTML, and you are done. Since the underlying program is written in JavaScript, the code is completely open. If the page developer has some understanding of JavaScript, all sorts of modifications are possible.

 

What about compatibility issues? Compatibility is a big issue in web-based programming. Kinetics Explorer has been designed with compatibility in mind. All of the “business” of the underlying components is isolated from the user interface. All browser-dependent features are isolated in just two packages, divgraph.js and divs.js. As browser rules change, minor modifications to these files will be necessary. However, there should be no problem with Netscape or Internet Explorer, Mac or PC, versions 4-7 at least, as these packages have been extensively tested on those platforms. (And if there is a problem, we can fix it!) All that should be required is a browser that uses Cascading Style Sheets (all browser versions since about 1998).

 

How far along is the project? Right now (1/26/03) the development stage has just begun. The basic JavaScript code is written and has been tested on a PC using Internet Explorer and Netscape. Fragmentary explorations have been written and can be demonstrated. The goal is to have content ready to go by April (2 months from this writing), and to have a preliminary web format worked out by the end of the summer. See the web site for updates.

 

KINETICS IN A NUTSHELL

 

Overall goals of kinetics:

 

  • to predict the course of chemical reactions
  • to optimize conditions for best (not necessarily fastest!) reaction
  • to understand what is going on “under the hood” in a chemical reaction

 

Basic sequence of investigation:

 

1.                  make observations of reactivity

2.                  gather time-based data

3.                  transform and tabulate the data

4.                  graph the data

5.                  determine underlying “empirical law” for the reaction

6.                  make predictions

7.                  test predictions

8.                  develop a hypothetical “mechanism” explaining the reaction
